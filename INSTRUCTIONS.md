# Инструкции по проекту

Эта памятка описывает шаги и допущения, по которым можно воспроизвести локальную
сборку и работу тулкита для офлайн-разбора B3 UMDF (MBO flavor).

## 1. Предварительные требования

- Python 3.12+ с `pip`, `cmake`, `ninja` (или GNU make) и компилятором C++20.
- Python-зависимости ставим через `python -m pip install -e .[dev]` после того,
  как соберётся нативный модуль.
- Локальный JDK (`third_party/jdk-21.0.2+13`) — его используем для запуска SBE
  codegen'а даже на машинах без системного Java.

## 2. Сторонние исходники

```
third_party/
  jdk-21.0.2+13/                 # локальный JDK для codegen'а
  simple-binary-encoding/        # клон форка Real Logic SBE
    sbe-all/build/libs/sbe-all-1.38.0-SNAPSHOT.jar
```

Репозиторий `simple-binary-encoding/` сейчас указывает на
`pcriadoperez/simple-binary-encoding` — он понадобился ради экспериментального
Python-генератора. Даже если мы используем только C++, клон оставляем: так
`scripts/generate_codecs.sh` берёт jar локально без выхода в интернет. При
необходимости обновления (например, до 1.37.1) достаточно выполнить:

```
cd third_party/simple-binary-encoding
export JAVA_HOME=/путь/к/jdk
./gradlew -x test :sbe-all:build
```

и направить `SBE_JAR` на полученный артефакт (или использовать уже
скомпиленный jar в `sbe-all/build/libs/`).

## 3. Генерация SBE-кодеков

```
./scripts/generate_codecs.sh
```

По умолчанию скрипт использует:

- схему `schema/b3-market-data-messages-1.8.0.xml`;
- выходной каталог `src/sbe-generated/b3_umdf_mbo_sbe/*.h`;
- jar `third_party/simple-binary-encoding/sbe-all/build/libs/sbe-all-1.38.0-SNAPSHOT.jar`.

Флаг `-Dsbe.generate.ir=true` заставляет SBE-tool писать файл
`schema/b3-market-data-messages-1.8.0.sbeir` — это слепок Intermediate
Representation. Его удобно держать рядом со схемой, если нужно прогнать
дополнительные утилиты SBE или сравнить кодогенерацию между версиями jar.
Поскольку это артефакт codegen'а, его можно смело удалять; при следующем запуске
скрипта он появится заново.

## 4. Сборка C++/Python

```
cmake -S . -B build -G Ninja \\
  -DPython_EXECUTABLE=$(which python)
cmake --build build
```

Эти команды соберут `_b3sbe_native`, который потом подхватывает `pip install -e .`
(через `pyproject.toml` + scikit-build-core). При желании можно сразу запускать
CLI из исходников, добавив `PYTHONPATH=src_autogenerated/python:build`.

## 5. CLI и рабочие сценарии

- `python -m b3sbe.cli inspect --input /путь/к/pcap`
- `python -m b3sbe.cli parse --input ... --output ...` — нативный парсер, пишет Parquet.
- `python -m b3sbe.cli build-updates ...` — строит `updates.parquet`.
- `python -m b3sbe.cli replay-book ...` — собирает книгу и BBO.
- `python -m b3sbe.cli debug-security ...` — отладка событий по конкретному инструменту.

## 6. Структура исходников

- Нативный код: `src/cpp/**`, сгенерированные заголовки — `src/sbe-generated/**`.
- Исследовательские чисто Python-модули находятся в `src/b3sbe_py/` и не входят в
  основной пайплайн.
- Табличные писатели не должны аллоцировать весь набор в память: данные надо
  стримить на диск (Parquet) сразу после декодирования.
