# Volatility Momentum Robot

Заметки по скрипту для онлайн-оценки моментума и волатильности по L1-стакану.

## Что делает скрипт
- Читает CSV L1 (как `calendar_spread/main`): сортирует по `packet_sending_time_ns`, вычищает подряд идущие дубликаты (`bid/ask price/size`), восстанавливает `mid_price`.
- Опционально приводит поток к указанному `frequency` (поддерживаются суффиксы `ms`, `s`, `min`, напр. `200ms`, `1s`, `0.5min`) через resample-last + forward-fill.
- Для `bid_price`, `ask_price`, `mid_price` запускает онлайн-алгоритм, который на каждом тике за `O(1)` выдаёт моментум и волатильность в миллисекундных окнах и меряет затраченное время вычисления (через `time.monotonic`).
- Результирующий DataFrame печатается (хвост), сохраняется PNG с двумя графиками (momentum/volatility) в файле `--output-plot`, и выводится статистика по времени (`mean/std` в миллисекундах).

## Онлайн-алгоритмы и почему `O(1)`
- **Momentum**: поддерживаем deque `(timestamp_ns, price)`. На каждом тике отбрасываем из головы всё, что «улетело» левее целевого времени `t - momentum_lookback_ms`, оставляя ровно последний апдейт не позднее этого горизонта. Одним параметром переключаем формулу: `price` даёт `Momentum(t) = P(t) - P(t - Δt)` (абсолютный импульс в деньгах/тиках), `log` — `Momentum(t) = ln(P(t)/P(t - Δt))` (относительный импульс, удобно сравнивать разные инструменты). Обе версии читают одно «опорное» значение и считают моментум за `O(1)`.
- **Volatility**:
  1. Второй deque хранит цены для поиска базы `t - volatility_frequency_ms`; как и в моментуме, двигаем голову вперёд, пока второй элемент не превысит целевой timestamp.
  2. Когда база найдена, считаем лог-доходность и кладём её в deque returns вместе с `timestamp_ns` конца интервала. Окно волатильности управляется по времени: выкидываем всё, что старше `t - volatility_lookback_ms`, и держим накопители суммы и суммы квадратов.
  3. Волатильность считаем только после того, как окно полностью прогрелось: ждём как минимум `ceil(volatility_lookback / volatility_frequency)` доходностей и требуем, чтобы самая старая return в очереди была не дальше чем на `volatility_frequency_ms` от левой границы окна (иначе данных недостаточно для выбранного горизонта). До этого момента в колонках остаётся `NaN`.
  4. Формула: `σ(t) = sqrt( Var[ r_{t-i} ], i=1..N )`, где `r = ln(P(t)/P(t-Δr))`, `Δr = volatility-frequency`, `N ≈ volatility-lookback/Δr`. Используем лог-доходности, чтобы симметризовать изменения и сделать итоговую волатильность аддитивной во времени. Сама дисперсия считается из накопителей суммы и квадратов, поэтому обновляется за `O(1)`.
- На выход пишем `*_momentum`, `*_volatility`, `*_calc_ms` (время в мс для каждого обновления).

## Запуск
```bash
python -m volatility_momentum_robot \
  data/orderbook/WDOF25/book_l1.csv \
  --frequency 400ms \
  --momentum-lookback 5s \
  --momentum-mode log \
  --volatility-lookback 1min \
  --volatility-frequency 1s \
  --output-plot volatility_momentum_robot/metrics.png
```

Параметры CLI:
- `l1_path` – путь к CSV с колонками L1 (`packet_sending_time_ns`, bid/ask price/size).
- `--frequency` – опционально, шаг дискретизации в формате `число + (ms|s|min)`; `100ms`, `0.5s`, `1min` и т.п.
- `--momentum-lookback` – горизонт моментума с теми же суффиксами (`ms`, `s`, `min`).
- `--momentum-mode` – `price` (абсолютный импульс `P(t)-P(t-Δt)`) или `log` (относительный `ln P(t)/P(t-Δt)`).
- `--volatility-lookback` – длина окна волатильности (`ms`, `s`, `min`).
- `--volatility-frequency` – шаг между точками для лог-доходностей (`ms`, `s`, `min`).
- `--output-plot` – путь к PNG (создаётся каталог, если нужно).

После выполнения в терминал попадает превью последней части таблицы, путь к PNG и статистика по времени расчётов для bid/ask/mid.
